# 🚀 1인 스타트업 실험실 운영 매뉴얼
> "블로그 하나를 하나의 '프로덕트'로 보고, 스타트업의 전체 사이클을 1인으로 시뮬레이션한다."

이 문서는 민지님의 블로그를 단순한 글쓰기 공간이 아닌, **데이터 기반의 프로덕트**로 성장시키기 위한 직군별 가이드라인입니다.

---

## 📅 1. Product Manager (기획 & 일정)
**목표**: 독자(유저)가 원하는 콘텐츠를 발굴하고, 블로그의 성장 방향성을 결정합니다.

*   **추천 툴**: [Notion](https://notion.so) (무료)
*   **할 일 (Workflow)**:
    1.  **Backlog 관리**: 떠오르는 아이디어를 노션 '아이디어 칸반 보드'에 저장.
    2.  **Sprint 설정**: "이번 주에는 기술 관련 글 1개, 에세이 1개를 발행하고 방문자 100명을 달성한다" 같은 1주일 단위 목표 설정.
    3.  **회고 (Retrospective)**: 매주 일요일, 목표 달성 여부와 배운 점 기록.

## 🎨 2. Product Designer (UX/UI)
**목표**: 유저가 머물고 싶은 아름답고 편안한 경험을 설계합니다.

*   **사용 툴**: [Figma](https://figma.com) (무료), Ghost Admin
*   **할 일**:
    1.  **썸네일 시스템화**: 글마다 썸네일 스타일이 중구난방이지 않도록, Figma로 나만의 '썸네일 템플릿' 제작.
    2.  **UI 커스텀**: `Ghost Admin -> Code Injection`을 통해 CSS로 폰트 자간, 행간, 버튼 컬러 등을 미세 조정하며 A/B 테스트.
    3.  **UX 실험**: "네비게이션 메뉴 순서를 바꿨을 때 클릭률이 달라질까?" 가설 세우고 검증.

## 💻 3. Developer (개발 & 배포)
**목표**: 안정적인 서비스를 제공하고, 자동화된 배포 파이프라인을 구축합니다.

*   **사용 툴**: VS Code, Terminal, GitHub, Vercel
*   **할 일**:
    1.  **로컬 개발**: `ghost start`로 로컬에서 기능/디자인 선 개발.
    2.  **CI/CD (배포)**: `npm run deploy` -> `git push`로 배포 자동화.
    3.  **테크 블로깅**: 본인이 겪은 기술적 문제(예: Vercel 도메인 연결 이슈)를 해결한 과정을 글로 남김 (이 자체가 훌륭한 개발 콘텐츠).

## 📊 4. Data Analyst (데이터 분석)
**목표**: 감(Feeling)이 아닌 데이터(Data)로 의사결정합니다.

*   **추천 툴**: [Google Analytics 4 (GA4)](https://analytics.google.com), Ghost Dashboard
*   **할 일**:
    1.  **데이터 심기**: Ghost Admin `Code Injection -> Site Header`에 GA4 추적 코드 삽입.
    2.  **지표 트래킹**:
        *   **Acquisition (획득)**: 사람들은 검색으로 들어올까, 트위터로 들어올까?
        *   **Engagement (참여)**: 어떤 글에서 체류시간이 가장 길었나?
        *   **Retention (재방문)**: 지난주에 온 사람이 이번 주에도 왔나?

## 📣 5. Marketer (그로스 & 홍보)
**목표**: 잠재 유저에게 도달하고, 팬(구독자)을 만듭니다.

*   **기본 기능**: Ghost의 **'Newsletter(뉴스레터)'** 기능 활용.
*   **할 일**:
    1.  **SEO 최적화**: 글 발행 전 `Metadata` 탭에서 구글 검색에 잘 걸리는 제목/설명으로 튜닝.
    2.  **뉴스레터 발송**: Ghost는 이메일 발송 기능이 내장되어 있습니다. (Mailgun 연동 필요). 구독자들에게 새 글 알림 보내기.
    3.  **채널 확장**: 링크드인, 트위터 등에 글 요약본을 올리고 블로그로 유입 유도 (Organic Marketing).

---
## 🔄 주간 루틴 예시 (Weekly Routine)
*   **월 (PM)**: 이번 주 주제 기획, 노션에 정리.
*   **화~목 (Maker)**: 글 작성(Writer), 썸네일 제작(Designer), 코드 수정(Dev).
*   **금 (Marketer)**: 배포 및 SNS 공유.
*   **토/일 (Analyst)**: GA4 들어가서 이번 주 어떤 글이 터졌는지 분석하고 다음 주 전략 수립.
